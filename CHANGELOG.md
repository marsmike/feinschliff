## [Unreleased]

### Added
- **Text-layout guardrails ŌĆö honest overflow gate, pack lint, collision prediction.** The slot-budget overflow gate now models node-level `size:`/`weight:` overrides, the real brand font face, and the emitter's autoshrink rescue (inset-aware, 10 pt floor) ŌĆö decompiled-pack decks are gated at their TRUE rendered size. One px/pt/EMU conversion source of truth (`feinschmiede.geometry.units`) + drift test; autoshrink ptŌåÆpx back-conversion fixed for non-13.33 in decks. Pack build writes `slot_warnings` (IMPOSSIBLE_BOX / NARROW_BOX / SLOT_BOX_OVERLAP, with role/decorative/grace precision) and `chrome_bboxes` front-matter; slotified title/body slots get an autoshrink floor (opt-out: tokens `"text-fit": {"autoshrink": false}`). Deck build predicts slot/slot + slot/chrome collisions (WARN severity, release 1) and warns OVERBUDGET against the now-honest measured `chars` budgets. **Migration note:** layouts using node-level `size:`/`weight:` overrides are now gated at their true rendered size ŌĆö strictly stricter gating; rendered output unchanged.
- **feinschnitt edit path.** `feinschnitt edit` CLI + `/edit` skill: plan-driven editing of pre-recorded talking-head footage into brand-themed cuts. Pipeline: word-timestamped Whisper transcription with a correction-memory layer ŌåÆ deterministic plan lint (per-kind contracts, density caps, text-placement floors) ŌåÆ speech-anchor alignment (extend-never-shorten, refrain-aware) ŌåÆ brand-themed Remotion engine (`edit-engine/` ŌĆö `hook_title`, `word_pop`, `stat_punch` templates + `StyleShowcase` golden comp) ŌåÆ preview/final render ladder with stage-keyed caching ŌåÆ voice-track remux verified bit-identical.
- **feinschnitt edit M2 template expansion.** Six new beat kinds ŌĆö `quote_pull` (typewriter takeover), `static` (full-frame image takeover), `image_card` (frosted b-roll overlay), `vertical_timeline` (rail-driven step takeover), `ratio_dots` (X-of-Y dot-grid overlay), `inline_chart` (self-drawing line-chart card) ŌĆö with per-kind lint contracts and a 9-kind `StyleShowcase`. Engine: a coverage underlay closes the boundary between back-to-back takeovers (no speaker flicker). Render: image beats are staged into the engine's `public/` and their mtimes join the render fingerprint. Align: `quote_pull` derives chars-per-second from the spoken anchor span and extends for a readable dwell. Lint: reading-time honors the stamped cps, kind-aware vertical ceiling for the chart card, and image-discipline checks (breathing room between image beats, extension-required paths).
- **feinschnitt edit M3 captions.** Word-synced captions generated from the transcript (`words.json`) ŌĆö never authored: Ōēż3-word chunks (portrait) / Ōēż5 (landscape) with per-word timing, rendered as a cardless bold-uppercase layer with current-word accent and measured-fit sizing. Beat-aware suppression: takeovers + lower-third text overlays silence captions in their window; visual overlays keep them unless the chunk semantically echoes the beat; a ┬▒0.8s echo pad around every beat drops chunks sharing meaningful (stopword-filtered, umlaut-folded) words with the beat's text. Plan config is `"captions": {"enabled", "emphasis"}` ŌĆö emphasis phrases (verbatim from the transcript) render accent + heavy weight, with unmatched/suppressed-phrase warnings on stderr.
- **Real-font text measurement** (`feinschmiede.text.measure`): fontconfig + PIL measure actual glyph widths when the brand font is installed; textfit, slot budgets, and orphan control use those widths automatically (char-ratio heuristics remain the fallback; `FEINSCHMIEDE_NO_REAL_METRICS=1` forces them). Over-wide words now model mid-word breaking instead of overflowing silently.
- **`italic:true`** ŌĆö new text primitive: accepted as a style-bundle flag and a per-node kwarg; italic runs are emitted directly in OOXML.
- **Native bullets** ŌĆö `bullet:true` / `bullet:"ŌĆō"` opt-in: emits `a:buChar` with a hanging-indent paragraph so bullet text lays out natively in PowerPoint/LibreOffice (no manual indent gymnastics required).
- **`--embed-fonts`** on `feinschliff build` / `feinschliff deck build`: embeds the brand's display and body faces (regular + bold) as OOXML `fntdata` parts so the deck renders correctly on machines where the font is not installed. Caller is responsible for font-license compliance (no `fsType` gate).
- **Four MS-gallery brand packs** in `feinschliff-extra` ŌĆö `annual-review`, `shapes`, `geometric`, `scientific`: Microsoft template-gallery ports with 13 slotified layouts each, machine-decompiled with the upgraded decompiler and measured against the source renders. Native chrome sidecars (`assets/native/`) and background artwork ship with the packs; MS content photos stay local, so picture slots fall back to the placeholder / image provider. `feinschliff-extra` now ships 14 packs.
- **feinschmiede umbrella mark.** Gem-cluster mark (`assets/feinschmiede-mark.svg` + dark variant) and a GitHub social-preview card; the brand gallery header now carries the mark and title.
- **Release zips.** CI builds an installable marketplace zip on every tag push.
- **Layout-picker profiles for decompiled brand packs (#60).** The slotify pass auto-generates picker frontmatter (role, ideal_count, data_band, comparison), slot roles with char capacities, image-slot provider query hints, fixed-chrome gating, and a per-pack `deck-map.yaml` ŌĆö without profiles the `/deck` picker can never choose a decompiled layout. The four MS-gallery packs ship generated profiles + deck-maps for all 52 layouts. Picture slots gain a `path:` + `query:` layered fallback (bound image > provider query > gem placeholder); the path/query mutex error is gone.

### Changed
- **Native autofit disabled on the real-metrics path**: when `feinschmiede.text.measure` resolves the brand font, the computed fit size is authoritative and PowerPoint's native autofit is suppressed (preventing PowerPoint / LibreOffice from re-deriving a different shrink). Heuristic path is unchanged. Emitter fit budgets (autoshrink, orphan control) are text-frame-inset-aware.
- **slot-overflow is FATAL by default**: `deck build` runs the static overflow gate unconditionally; a slot-overflow error aborts the build before `compile_slide()` is called. `--strict-static` now only promotes WARN-severity defects to build-blockers (slot-overflow is already fatal). `--skip-content-lint` no longer bypasses geometry checks.
- **Picture slots letterbox by default**: the default crop mode for non-cover picture slots is now contain (letterbox) instead of fill (stretch-to-cover). Aspect ratio is preserved; the picture is centred in the slot.
- **Semi-opaque text blends toward brand paper**: run alpha is approximated by blending toward the brand's `paper` token colour instead of white ŌĆö dark-brand decks no longer get inverted-contrast ghost text.
- **Slot-budget pxŌåÆEMU scale derived from `slide.width_emu` token**: decompiled-brand packs now calculate overflow budgets against the real slide dimensions (previously they used a wrong default, causing silent mis-scaled overflow predictions).
- **Diagram render cache is real**: unchanged diagrams skip re-render ŌĆö artifacts are named by content hash and reused across builds.
- **`effect-allow` sanitation marker is XML-namespaced** (`urn:feinschliff:emit`): the chrome scanner now recognises the namespaced form on gradient fills as well as plain shapes; legacy bare-attribute form is still honoured on read for backward compatibility.
- **Font metrics are brand data (#61).** Glyph-width ratios for the slot-budget / verify-static predictors are no longer hardcoded ŌĆö packs declare a `font-metrics` block in tokens.json, registered at compile time via the new `register_font_metrics` hook.
- **Decompiler fidelity (#51ŌĆō#56).** Complex shapes, template images, graphicFrames, and groups carry natively instead of as rasterised pictures; text slots extend into available clearance for `/deck` overflow; per-master theme resolution with palette/background capture; inherited text colour (on-shape + placeholder), z-order, caps/bold, and vertical-alignment inheritance; clrMap resolution, placeholder-idx pairing, table-style + bg-image carry, `showMasterSp`; repair-error ids, sidecar payloads, slotify / hidden-shape / slot-blanking fixes; the brand theme font is carried into the generated pack.
- **Discovery: working tree beats installed plugin (#59).** Brand/layout/provider discovery precedence is now `bundled > env > cwd-dev > plugin > user`, and cwd-dev also detects sibling `feinschliff-*` plugin dirs (`brands/`, `layouts/`, `feinschliff_providers/`) in a checkout ŌĆö `feinschliff-extra` and private packs are visible without `FEINSCHLIFF_BRAND_PATH`, and a stale marketplace copy of a same-named brand no longer shadows the working tree. The office launcher likewise registers brands + layouts from sibling `feinschliff-*` plugins by glob instead of a hardcoded list.
- **One shared verify-map loader (builder)**: seven per-script parsers that each re-read and re-parsed the brand verify-map are replaced by a single loader; `brand_verify_loop` routes LibreOffice and pdftoppm through the canonical profile-isolated helpers and imports the visual-diff logic in-process (previously a raw `soffice` + subprocess chain that could silently fail under parallel runs); `pdf_to_pngs` gained a `scale_to` parameter.
- **Manifest drift checks ŌĆö one plugin registry**: `gen_launchers` now maintains a single registry that simultaneously verifies `marketplace.json`, `plugin.json` names, `uv` workspace members, and the four lists in `ci.yml`; marketplace copy corrected from "10 themes" to the 14 that actually ship.
- **Picture-provider / asset-lock / HTTP-materialise subsystem extracted from `pptx_emit`**: the image-materialise logic (~385 lines) has moved to `feinschliff/io/image_materialise.py`; `pptx_emit` drops from 2,698 to 2,313 lines.

### Fixed
- **`deck apply-fixes` accepts engine severity vocabulary**: `deck verify-static --json` emits severities like `error`/`warning`/`info`; `apply-fixes` was silently dropping every defect it received because it expected a different key shape ŌĆö the documented verifyŌåÆapply round-trip never actually worked.
- **Diagram wireframe text-width uses `CHAR_WIDTH_EM`**: the per-character width constant in the wireframe estimator was a stale pixel literal that underestimated scaled title widths by ~3.4├Ś; it now reads the shared `CHAR_WIDTH_EM` metric so display-size titles no longer overflow their wireframe boxes.
- **Decompile: charts in shared graphic frames and scaled-group offsets**: charts that share a single graphic-frame part within one frame no longer drop the second reference (only the first chart was captured). `graphicFrame` elements inside scaled groups no longer crash ŌĆö the offset was returned as an 8-tuple; the decompiler now unpacks all fields.
- `--brand` on the CLI now overrides a deck's `@brand` directive (the brand-not-found diagnostic is kept).
- The blank base template is excluded from the brand gallery.

### Removed
- **`feinblick`** ŌĆö the codebase-intelligence plugin moved to the `agentic-toolkit` marketplace as `codescan` (an internal static-analysis / audit tool, unrelated to media creation). Its self-audit config (`feinblick.toml` + the `.feinblick/` baseline) is dropped from this repo too.
- **`feinklang-consumer`** ŌĆö the throwaway Phase-0 cross-plugin smoke-test plugin (already de-listed from the marketplace).

## [0.2.0] - 2026-05-21

### Added
- **Deck-wide layout budget planning** (`feinschliff/lib/layout_budget.py`) ŌĆö `feinschliff deck plan-skeleton` now runs a second pass on top of `lib.layout_picker` that re-ranks per-slide candidates with a usage-budget bonus (+1.5 for a never-used layout, +0.75 after one use, +0.5 after two, etc.). Eligible-but-overlooked layouts (`vertical-bullets`, `pyramid`, `funnel`, `four-column-cards`, ŌĆ”) now surface across long decks instead of the same two or three winners repeating. The bonus is calibrated below the role-match weight (+3), so a strong affinity match still wins on the first use; coverage only kicks in among co-eligible candidates. Singleton layouts (`agenda`, `end`, `full-bleed-cover`) are exempt. Each skeleton slide records the picker + budget rationale under `_meta.layout_rationale` for debugging. Smoke test on a 17-slide mixed deck went from ~7 distinct layouts (baseline) to 15. Covered by `tests/test_layout_budget.py` (14 tests pinning both halves of the contract: spreads usage across co-eligible layouts; never overrides a strong affinity match like a `risk-matrix` fingerprint).
